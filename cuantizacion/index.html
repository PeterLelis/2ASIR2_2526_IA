Optimización de LLM locales

Optimización y cuantización del modelo Phi-3-mini utilizando llama.cpp sobre procesadores Apple Silicon (Mac Mini).

Fundamentos técnicos

¿Qué es la cuantización?

La cuantización es el proceso de reducir la precisión numérica de los parámetros (pesos) de un modelo. En lugar de almacenar cada peso en formatos de alta precisión como FP32 (32 bits), los convertimos a formatos menores como INT4 (4 bits).

Esto es similar a "redondear" valores decimales complejos a números enteros. Aunque se pierde algo de precisión, el modelo sigue representando casi el mismo valor, pero ocupando una fracción del espacio original.

Compresión

Reducción del tamaño del archivo en más de un 70%.

Velocidad

Inferencia más rápida gracias a cálculos con enteros.

Memoria RAM

Permite ejecutar modelos gigantes en solo 8GB o 16GB de RAM.

Accesibilidad

Democratización de la IA fuera de los centros de datos.

Paso 01

Entorno y Dependencias

Como administradores de sistemas, el primer paso es garantizar el aislamiento del proyecto. Utilizamos un entorno virtual de Python (venv) para evitar conflictos con las librerías del sistema macOS.

Instalamos las dependencias críticas como transformers y torch, necesarias para que el script de conversión pueda interpretar los tensores del modelo original.

Terminal - setup.sh
mkdir ejercicio-cuantizacion && cd ejercicio-cuantizacion
git clone https://github.com/ggerganov/llama.cpp.git
python3 -m venv venv && source venv/bin/activate
pip install -r llama.cpp/requirements.txt

Preparación del taller de trabajo

Compilación optimizada con CMake

Paso 02

Compilación para Metal

Para aprovechar la GPU unificada del Mac Mini, es imperativo compilar los binarios de llama.cpp localmente. Utilizamos CMake, que detecta automáticamente la arquitectura ARM de Apple Silicon.

Este proceso genera ejecutables capaces de usar la aceleración de hardware 'Metal', lo que reduce drásticamente la latencia en la generación de texto.

Terminal - build_cmake.sh
cd llama.cpp && mkdir build && cd build
cmake ..
cmake --build . --config Release -j
Paso 03

Descarga del Modelo (Git LFS)

Los modelos de IA se almacenan como archivos binarios masivos. Para descargarlos, necesitamos Git Large File Storage (LFS), que gestiona estos archivos de varios gigabytes de forma eficiente.

Terminal - download_model.sh
git clone https://huggingface.co/microsoft/Phi-3-mini-4k-instruct
cd Phi-3-mini-4k-instruct && git lfs pull

Obtención de pesos en alta precisión FP16 (~7.5 GB).

Etapa Final

Proceso de Cuantización Activa

Mapeo de tensores y reducción de parámetros mediante la técnica K-Quants.

Método Bits Explicación del Nivel Tamaño
F16 (Original) 16-bit Máxima calidad, sin compresión. Uso exclusivo en servidores. 7.64 GB
Q4_K_M (Utilizado) 4-bit Equilibrio perfecto. La inteligencia es indistinguible de la original. 2.39 GB
Q8_0 8-bit Casi sin pérdida, pero archivos aún pesados. ~4.0 GB

ANÁLISIS DE TENSORES EN TIEMPO REAL

Terminal - quantization.sh
./llama-quantize phi-3-mini-f16.gguf phi-3-mini-q4.gguf q4_k_m

VALIDACIÓN Y CIERRE DE ARCHIVO GGUF

El script analiza los rangos de valores de los pesos originales y los mapea al rango mucho más pequeño de 4 bits. Al finalizar, guarda la "escala" necesaria para realizar el cálculo inverso aproximado durante la ejecución.

Benchmarks de Infraestructura

1. Eficiencia de Almacenamiento

Al reducir la precisión de 16 a 4 bits, hemos liberado casi 5.25 GB de almacenamiento. Esto permite que el modelo sea mucho más portátil y rápido de cargar en la RAM.
